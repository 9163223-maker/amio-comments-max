# Changelog

## 14.15.SP3.14.6.23

- Исправлено позиционирование нативного меню прикрепления файла: file input теперь физически находится на нижней скрепке.
- Убран программный `attachmentInput.click()`, который в iOS/MAX WebView открывал picker в середине экрана.
- Исправлено отображение фото в комментариях: теперь фото показывается как изображение, а не как файл.
- Для вложений комментариев добавлен локальный `previewUrl` в `/public/comment-uploads/...`.
- Native MAX payload (`payload.photos`) сохраняется отдельно от preview и без base64 в `store.json`.
- `store` и `commentService` сохраняют nested `payload.photos` и `previewUrl` безопасно.
- После пересылки старого поста бот возвращает пользователя в исходный раздел: комментарии, посты, кнопки, статистика или подарки.
- Добавлен глобальный стресс‑тест меню `test/global-menu-stress.mjs`.
- Runtime поднят до `14.15.SP3.14.6.23`.

## 14.15.SP3.14.6.18

- Убрана двойная загрузка вложений.
- Добавлен upload в MAX с Authorization.
- `dataUrl/base64` не сохраняется в `store.json`.

## 14.15.SP3.14.6.23

- Добавлен полноэкранный предпросмотр медиа перед отправкой комментария.
- Фото/видео комментарии теперь отправляются через сценарий: выбор → preview → подпись → отправка.
- Фото отображается в комментариях как картинка, а не как файл `IMG_....jpeg`.
- Вложения сохраняют лёгкий `previewUrl` и нативный MAX `payload`; base64 не пишется в store.
- Счётчик комментариев на кнопке поста в канале обновляется после создания/удаления комментария.
- Repatch счётчика вынесен в debounce/background и не запускается на лайках/реакциях комментариев.
